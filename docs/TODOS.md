## Critical (fix before writing code)

- [x] **`focusOrder` field — dual ordering system** — `focusOrder` (Today position) initializes to `order` value when `isFocused` is set to true. Today DnD writes `focusOrder` ONLY. `order` is written only by Inbox/Project DnD. Fractional indexing rebalancing logic must cover both fields.
  - 구현: `task_model.dart` 필드 추가 + `task_repository.dart` `setFocused()` / `updateFocusOrder()` / `rebalanceOrder(useFocusOrder)` 분리

- [x] **`importBatch: true` flag on tasks during Google Tasks import** — Add `importBatch: bool?` field to task schema. `ImportService` sets this to `true` on every task written during import. Cloud Function `onTaskWrite` checks: if `importBatch == true`, bail immediately (skip calendar sync). After import completes, `ImportService` triggers a single batch calendar sync for all tasks with `dueDate` set.
  - 구현: `task_model.dart`에 `importBatch: bool?` 추가

- [x] **`completionNonce` for repeat task idempotency** — Add `completionNonce: string?` field to task schema. When completing a repeat task, generate a UUID and set `completionNonce`. Cloud Function checks: if a sibling task with `parentNonce == thisTask.completionNonce` already exists, skip creating next occurrence.
  - 구현: `task_model.dart`에 `completionNonce: String?` 추가 + `task_repository.dart` `completeRepeatTask()` WriteBatch 구현

- [x] **Calendar sync silent failure gap** — Add `calendarSyncStatus: "ok" | "failed"` field to task document. Cloud Function sets to `"failed"` after 3 retries. `calendarSyncStatusProvider` watches this field and surfaces a retry banner or badge in Settings.
  - 구현: `task_model.dart`에 `calendarSyncStatus: String?` 추가

- [x] **Repeat task completion atomicity** — Use `WriteBatch` for repeat task completion: `batch.update(taskRef, {completed: true, completedAt: now})` + `batch.set(newTaskRef, nextOccurrence)` committed atomically.
  - 구현: `task_repository.dart` `completeRepeatTask()` — WriteBatch로 완료 + 다음 occurrence 원자적 처리

- [x] **Bulk import + LocalCalendar ANR** — During Google Tasks import, do NOT call `LocalCalendarService.writeEvent()` per task. Add `_importInProgress` flag to `ImportService`. After import completes, batch-trigger calendar sync for all dueDate tasks using `compute()` or background isolate.
  - 설계 확정: `importBatch=true` 플래그로 Cloud Function 동기화 차단, 완료 후 일괄 트리거. ImportService 구현 시 반영 필요.

- [x] **`googleCalendarToken` in user document** — Move to `users/{uid}/private/googleCalendar` subcollection. Firestore field-level rules don't exist; the whole user document read exposes the refresh token to the client. Subcollection can have its own uid-match rule.
  - 구현: `google_calendar_credential_model.dart` 생성 + `firestore.rules`에 `users/{uid}/private/{doc}` 별도 규칙 추가. `user_model.dart`에는 토큰 필드 없음.
